# 回归测试协议（制度化·上级裁决落地）

本文件固化本工程回归/审计的执行制度，供后续所有轮次遵守。

## 协议1·验案锚可见性（[待裁决-验案锚可见性]结案）

**裁决（上级批准）**：引擎内各方证条目的 [HXS验案] 锚**全部保留**——它们是合法的
誊写知识（胡老验案原文，供辨证追溯与判据佐证），**不作执行期屏蔽**。

**配套审计协议（制度化，防"用答案证明答案"污染通过率）**：
- 冷启动/遮盲审计中，**若某测试案在引擎内有其自身的验案锚**（即该案原文已作为某
  方证条目的 [HXS验案] 挂入引擎），该案在审计中**标 ※ 单列**。
- **※案不计入冷启动通过率的分子，也不计入分母。** 通过率只由"引擎内无自身验案锚"
  的干净案构成。
- ※案仍照常走查并记录结果（用于观察一致性），但其"命中"不得充作引擎判据能力的
  证据——因为冷执行器读引擎时可见该案答案。
- 守卫案（G1-G15）本就内嵌金标准，由 R14 管辖（禁引守卫块作判据）；※规则是对
  **普通方证条目验案锚**的平行制度，二者互补。

**识别方法**：审计抽样后，对每个抽中案，按病历号/就诊日期/特征短语三重 grep 引擎
文本；命中即标 ※。（已实现于 A0 审计脚本，本轮沿用。）

## 协议2·两数/多数并列（诚实基线）

毕业考/未遮盖通过率与遮盖审计真值**并列呈现，旧数不删**。修复轮后新增第三数
（未遮X% / 首审Y% / 修复后Z%），累积并列。禁止用新数覆盖旧数或加缓冲性解释。

## 协议3·分层抽样与换种子复审

- 抽样：103案审计框按七方族分层，随机种子记录在案。
- 修复后复审：**换种子**（避免固定样本过拟合），n=30，**强制含全部已知W案**
  +随机补足至30，※规则生效。

## 协议4·OCR产出纪律（P7-B起生效）

- 所有OCR产出的**处方剂量**标 [OCR未校]，未经人工核对**不得作守卫金标准**。
- 判据挂载须**原文锚 + 抽样人工核验**；OCR噪声条目**宁弃勿猜**。
- 冯世纶自著发挥标 [冯] 层级，与 [HXS] 区分。

## 协议5·冷执行器纪律（既有，归档于此）

- 所有回归/审计由全新冷会话（无迭代记忆）执行，主会话仅设计+揭盲比对。
- R11截断自检 / R1闭卷 / R14守卫禁引 / R15判例通读，冷会话须全程遵守。
- 会话用量限额中断时断点续跑，不并行烧窗口（串行或≤2并发）。
